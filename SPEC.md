## 1. コンセプト

**「クイズに正解すると弾が出て敵を撃つ」**

スペースインベーダー型のシューティングゲームにクイズを組み合わせたアーケード風ゲーム。  
プレイヤーは十字ボタン（Dパッド）で3文字のかな回答を入力し、正解すると自機が敵に向かって弾を発射する。  
敵を全滅させてWaveをクリア、全Waveを突破するとステージクリアとなる。

**ターゲット:** ブラウザで動作するカジュアルゲーム。スマートフォンタッチ操作にも対応。

## 2. ゲームフロー

```
タイトル画面
    ↓ タップ/ボタン
ステージ選択画面（5×5グリッド）
    ↓ ステージ確定
Wave開始シーケンス（READY → GO！）
    ↓
playing（クイズ回答 → 弾発射 → 敵撃破）
    ↓ 敵全滅
    ├─ 次のWaveへ（Waveループ）
    ├─ 固定ステージ最終Wave完了 → ステージクリア画面
    └─ 敵が防衛ライン突破 or HP0 → ゲームオーバー画面
ゲームオーバー / ステージクリア
    ↓ タップ/ボタン
タイトル画面（ループ）
```

## 4. クイズシステム

### 4-1. 基本ルール

- 問題はすべて**かな3文字**で答える形式
- Dパッド（上/下/左/右）の4ボタンにそれぞれ1文字が割り当てられる
- プレイヤーは3回ボタンを押して回答を構成する
- 3文字入力完了で自動判定（確定ボタン不要）
- 中央ボタン（×）で入力リセット可能

### 4-2. 文字の配置ルール

各問題には `chars` フィールド（4文字配列）が定義されており、そこからDパッドに割り当てる。

```
chars = ["り", "ん", "ご", "か"]  → 4ボタンにシャッフルして配置
```

- `chars` が4文字未満の場合は `kanaPool` からランダム補充
- **正答に必要な文字は必ずDパッドに含まれる**（安全策で補充）

### 4-3. タイマー

- 制限時間はウェーブ設定の `timerSec` で決定
- 残り時間はゲージバーでビジュアル表示
- 時間切れは不正解扱い（→武器ランクダウン）

### 4-4. 正解・不正解の処理

| 結果 | 処理 |
|------|------|
| 正解 | タイミング評価を算出 → 弾発射 → コンボ+1 |
| 不正解 | コンボリセット → 武器ランクダウン（-20点相当の内部ペナルティ） |
| タイムアップ | 不正解と同じ |

### 4-5. タイミング評価（gaugeRatio）

正解時に「残り時間 / 制限時間」比を計算してランクを決定。

| 比率 | ランク | スコア倍率 |
|------|--------|-----------|
| ≥ 0.9 | ACE | ×3.0 |
| ≥ 2/3 | PERFECT | ×2.0 |
| ≥ 1/3 | GREAT | ×1.5 |
| > 0 | HIT | ×1.0 |
| = 0 | miss | ×0（弾は出ない） |

## 5. 武器コンボシステム

### 5-1. 武器ランク

コンボ数に応じて武器が強化される。

| ランク | 武器名 | 次のランクまで | 弾のビジュアル |
|--------|--------|--------------|--------------|
| 1 | LASER | コンボ3 | 緑レーザー |
| 2 | MISSILE | コンボ3 | オレンジミサイル |
| 3 | BEAM | コンボ4 | 青白ビーム |
| 4 | NOVA | 上限なし | 紫エネルギー弾（超兵器） |

### 5-2. 撃破数（killCounts）

正解1問で倒せる敵の数はランクとタイミングで変動。

| ランク | HIT | GREAT | PERFECT | ACE |
|--------|-----|-------|---------|-----|
| LASER | 1 | 2 | 3 | 5 |
| MISSILE | 2 | 3 | 4 | 6 |
| BEAM | 3 | 5 | 6 | 8 |
| NOVA | 4 | 7 | 10 | 15 |

- 不正解でランクが1下がる
- ランク4の状態でランクダウンするとランク3に戻る

### 5-3. チャージショット

- 正解連続時、一定条件でチャージ演出が発動
- チャージ中に正解すると特大の弾が出る（サイズ拡大・威力増加）

## 6. 敵・ボス

### 6-1. 通常Wave（type: 'normal'）

スペースインベーダー型の敵フォーメーション。

| パラメータ | 説明 |
|-----------|------|
| `cols` | 列数（最大8） |
| `rows` | 行数（最大5） |
| `speed` | 横移動速度 |
| `descent` | 降下速度（基本） |

**降下速度の加速ロジック:**
- 敵が `DANGER_Y`（画面55%地点）に近づくと速度が加速（×1.65）
- 防衛ライン（DEFENSE_Y）を越えると減速（×0.35）

敵のピクセルアートは3種類のパターンからランダム選択。色は5色からランダム。

### 6-2. ボスWave（type: 'boss'）

| パラメータ | 説明 |
|-----------|------|
| `hp` | 体力（正解1問ごとにkillCounts分減少） |
| `width` / `height` | サイズ |
| `speed` | 横移動速度 |
| `descent` | 降下速度 |
| `design` | 外観 `'fortress'` / `'wing'` / `'core'` |
| `color` | 色コード |
| `pattern` | 特殊攻撃の配列 |
| `patternProb` | 特殊攻撃の発動確率（0〜1） |

### 6-3. ボスの特殊攻撃パターン

| パターン | 効果 |
|---------|------|
| `scramble(n)` | Dパッドのn個のボタンの文字を `?` に隠す |
| `blind(n)` | 問題文のn文字をランダムに `？` でマスクする |
| `marquee(n)` | 問題文をn秒かけてスクロール表示（視認を難しくする） |

## 7. スコアシステム

- 基本スコア = `killCounts × タイミング倍率 × ステージ係数`（詳細は実装依存）
- ステージごとにハイスコアを `localStorage` に保存
- ゲームオーバー・クリア時にNEW RECORD判定を表示

---

## 8. ステージ定義システム

### 8-1. ステージ種別

| 種別 | 説明 |
|------|------|
| **固定ステージ**（`waves` 配列） | 全WaveをJSオブジェクトで手動定義。クリア条件あり |
| **Infinityモード**（`mode: 'infinity'`） | Wave番号から動的にWaveを生成。終わりなし |
| **Legacyモード** | 旧来の数式ベース生成（廃止予定） |

### 8-2. ステージ選択UI

- 5×5の25スロットグリッド
- 各スロットに `slot` 番号でステージを割り当て
- 固定ステージのクリア済みフラグは `localStorage` で保持し、"CLEAR"バッジを表示

### 8-3. Infinityモード Wave生成ルール

優先順位: **exactWaves > rules > defaultWave**

| 設定 | 説明 |
|------|------|
| `exactWaves` | 特定Wave番号への完全固定設定（チュートリアル序盤など） |
| `rules` | `every: n`（n倍数のWave）に適用するルール。複数ヒット時は`priority`が高い方が優先 |
| `defaultWave` | どれにも当てはまらない通常Wave |

### 8-4. カーブ計算（difficulty scaling）

Infinityモードではパラメータを式で自動計算できる。

| タイプ | 式 |
|--------|----|
| `constant` | 固定値 |
| `linear` | `base + wave * perWave`（+ `perBoss * bossIndex`） |
| `step` | `base + floor(wave / every) * add` |
| `stepValue` | Wave番号が`minWave`を超えたら対応する値に切り替え |

すべての計算結果には `min` / `max` でクランプ可能。

## 9. クイズ問題データ

### 9-1. ジャンル一覧

`くだもの` / `いきもの` / `しぜん` / `くらし` / `あそびと文化` / `たべもの` / `からだ` / `がくもん` / `サブカル` / `ちめい` / `げいのう` / `にっち` / `mixed`

`mixed` はフィルタなし全ジャンルから出題。

### 9-2. 難易度レベル

| level | 目安 | 割合（設計値） |
|-------|------|--------------|
| 1 | やさしい | 60% |
| 2 | ふつう | 30% |
| 3 | むずかしい | 10% |
| 4 | とてもむずかしい | 少数 |

### 9-3. 問題データ構造

```js
{
  level: 1,
  question: "赤くて丸い果物は？",
  answer: "りんご",    // 必ず3文字
  chars: ["り", "ん", "ご", "か"]  // Dパッドに表示する4文字
}
```

## 14. 拡張可能性（ブレスト用メモ）

- **新ジャンル追加**: `questions.js` にデータを追加し `stage_defs.js` の `genres` に指定するだけ
- **新ボスパターン**: `quiz.js` の `_applyPatternEffects` に新パターンを追加
- **新武器ランク**: `COMBO_RANKS` 配列に追加（NOVA以降）
- **マルチ回答方式**: 現在3文字固定。2文字・4文字問題への対応は`quiz.js`の`inputPos`上限変更が必要
- **新ステージモード**: `mode` フィールドに新値を追加して `_getWaveDef()` で分岐
- **スコアランキング**: `localStorage` ベースのハイスコアは既存。サーバー送信は未実装
- **BGM/SE**: 現在は未実装（SEなし）
